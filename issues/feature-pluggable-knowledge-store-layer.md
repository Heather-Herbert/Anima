# Feature: Pluggable Knowledge Store Layer (Local and Enterprise Backends)\n\n## Summary\nImplement a Knowledge Store abstraction that supports both:\n- Local embedded storage (e.g., SQLite, DuckDB, or similar; ships by default)\n- External database adapters (e.g., PostgreSQL, Redis, etc.) for enterprise/scaling scenarios\n\n## Motivation\n- Make memory and long-term context persistent and queryable without external dependencies for most users\n- Allow teams/orgs to migrate to enterprise DB backends for scale, HA, integrations, and compliance\n- Simplify onboarding\n\n## Requirements\n- [ ] Abstract a "Knowledge Store" interface (CRUD, vector search, key/value, artifact storage)\n- [ ] Implement local-first backend (SQLite or DuckDB, stored under Memory/)\n- [ ] Add pluggable backend support (users can specify Postgres/Redis URLs in config)\n- [ ] Clearly document the supported features and limitations of each backend\n- [ ] CLI/config file option to select backend (`--memory-backend sqlite|duckdb|postgres|redis`)\n- [ ] Migration utility: Local <-> Enterprise backend (export/import)\n- [ ] Backward compatibility: Old JSON memory files are auto-imported\n\n## Out of Scope (for MVP)\n- No UI for database management (CLI/config only)\n- No support for distributed consensus (single instance per DB connection)\n\n---\n\n## Related to enterprise agent architecture [video]:\n- Unlocks "Knowledge Store Layer" foundation\n- Enables persistent, editable, and scalable artifacts\n\n---\n**Priority:** High\n**Labels:** enhancement, architecture, knowledge-store, enterprise
